# ココロバランス 要件定義書

**バージョン:** 1.0  
**作成日:** 2026-06-29  
**ステータス:** ドラフト

---

## 1. プロジェクト概要

### 1.1 プロダクト名

**ココロバランス（KokoroBalance）**

### 1.2 キャッチコピー

> 心の資産を分散しよう。  
> あなたの心は、何で満たされていますか？

### 1.3 プロダクトコンセプト

「自分の心が何によって支えられているか」を可視化し、依存の偏りを防ぎ、長期的に心を安定させるための**自己マネジメントツール**。

メンタルヘルスが悪化してから対処するのではなく、**悪くならないように予防する**アプリ。投資の分散理論をメンタルヘルスに応用した「心のポートフォリオ」という独自の考え方を中核に置く。

---

## 2. 解決する課題

### 2.1 課題の背景

人は恋人・推し・仕事・SNS・お金など特定の対象に依存しやすい。その対象が不安定になった時、メンタルも大きく崩れる。

一方、**複数の心の支えを持っている人ほど精神的に安定しやすい**という事実がある。

しかし、「自分が何に依存しているか」は意外と気づきにくい。

### 2.2 既存サービスとの差別化

| 比較軸 | 既存の気分記録アプリ | ココロバランス |
|---|---|---|
| 記録対象 | 感情・気分の結果 | 心を満たす"資産"の分布 |
| アプローチ | 事後対処型 | 予防型 |
| 分析視点 | 今日の状態 | 長期的な依存傾向 |
| アクション提案 | なし or 汎用的 | 偏りに応じた具体的行動提案 |

---

## 3. ターゲットユーザー

### 3.1 メインターゲット

- **年齢:** 20〜40代
- **属性:** 社会人、恋愛中、推し活、仕事中心の生活、一人暮らし

### 3.2 サブターゲット（ペルソナ）

- メンタルが落ち込みやすい人
- 仕事が忙しく自分を振り返る余裕がない人
- 恋人への依存が強くなりやすい人
- SNS疲れを感じている人

---

## 4. コアUX（ユーザー体験フロー）

```
毎日1分。今日、自分の心は何で満たされた？
        ↓
    カテゴリを選択・満足度を入力
        ↓
    心のポートフォリオが更新される
        ↓
    偏りが可視化される
        ↓
    AIが改善アドバイスを提示
```

## 5. 機能要件

### 5.1 MVP（フェーズ1）

#### 5.1.1 ユーザー登録・プロフィール設定

- メールアドレス／SNSアカウントによるサインアップ・サインイン
- ニックネーム設定
- 初回オンボーディング：心のカテゴリ選択

#### 5.1.2 心のカテゴリ設定

初期カテゴリとして以下を提供。ユーザーは有効にするカテゴリを選択し、自由入力でカスタム追加もできる。

| 大カテゴリ | 小カテゴリ例 |
|---|---|
| 人 | 恋人、家族、友達、同僚 |
| 趣味 | ゲーム、音楽、映画、読書、カフェ |
| 推し | アイドル、アーティスト、VTuber |
| 自己成長 | 勉強、筋トレ、副業、資格 |
| 健康 | 睡眠、運動、散歩、温泉 |
| 仕事 | 達成感、給料、人間関係 |
| お金 | 投資、貯金 |
| その他 | 自由入力 |

#### 5.1.3 毎日の記録

- 今日満たされたカテゴリをチェックボックスで複数選択
- 各カテゴリに満足度スコア（0〜100）または星評価（1〜5）を入力
- マイナス記録：「恋人と喧嘩 -60」のようなネガティブイベントも記録可能
- 記録は1日1回を基本とするが、1日複数回の更新も許容
- プッシュ通知によるリマインド（時間はユーザーが設定）

**記録例:**
```
恋人：80
筋トレ：40
仕事：70
推し：30
```

#### 5.1.4 心のポートフォリオ表示（メイン画面）

- 円グラフで各カテゴリの割合を表示（過去30日間の集計）
- バーグラフ（ポートフォリオ表示）でも切り替え可能
- 集計期間：1週間 / 1ヶ月 / 3ヶ月 を切り替え可能

**表示例（円グラフ）:**
```
恋人  45%
仕事  25%
趣味  15%
家族  10%
運動   5%
```

#### 5.1.5 偏りアラート

- 単一カテゴリが60%以上を占める場合に警告表示
- アラートメッセージ例：
  > ⚠ 恋人への依存が高くなっています  
  > もし恋人と喧嘩した場合、心が大きく揺れる可能性があります。  
  > 別の楽しみを育てましょう。

#### 5.1.6 週間レポート

- 毎週月曜日に自動生成（またはユーザーが任意タイミングで確認）
- 内容：
  - 先週のカテゴリ別割合サマリー
  - 前週比の変化
  - AIによる来週へのアクション提案

**レポート例:**
```
今週は
・仕事が多め（45%）
・睡眠不足が記録されています
・友達との交流ゼロ

来週のおすすめ
・散歩（15分でもOK）
・読書（1冊完読しなくてよい）
・友達との食事を1回予定に入れる
```

#### 5.1.7 AIコーチ（チャット形式）※Proプラン限定

- ユーザーが近況をテキスト入力
- AIがポートフォリオのデータをもとに具体的なアドバイスを返す
- 無料ユーザーには「AIコーチはProプランで利用できます」の案内を表示し、アップグレードへ誘導

**会話例:**
```
ユーザー：最近恋人しか楽しみがない
AI：最近は仕事・趣味・友達の割合が減っています。
    今週は友達とご飯に行く予定を一つ入れてみませんか？
```

#### 5.1.8 心の残高（デイリースコア）

- その日の記録をもとにプラス・マイナスを集計
- 日次スコアをホーム画面に表示

**表示例:**
```
今日の心の残高
+30 筋トレ
+20 友達
+50 仕事
─────────────
合計 +100

（先週：仕事-60（喧嘩） などマイナスも記録）
```

#### 5.1.9 心の資産スコア

各カテゴリに累積的な「資産スター」を付与。継続的に記録することで資産が積み上がる。

```
心の資産
恋人   ★★★★★
趣味   ★★★
友達   ★★★★
健康   ★★★★
```

### 5.2 フェーズ2（将来機能）

#### 5.2.1 ライフイベントシミュレーション

「もし仕事を辞めたら」「恋人と別れたら」などの仮定を入力すると、心のポートフォリオへの影響を可視化する。

#### 5.2.2 心の目標設定

「運動の割合を10%→15%にしたい」などの目標を設定し、達成に向けた行動をAIが提案する。

#### 5.2.3 ウェアラブル連携

睡眠・歩数・運動量をデバイスから自動取得し、「健康」カテゴリを自動更新する。

#### 5.2.4 振り返りジャーナル

心が大きく動いた出来事とその理由を記録。AIが長期的な傾向を分析する。

#### 5.2.5 匿名コミュニティ

「心の資産を増やす工夫」を匿名でユーザー間で共有できる場を提供する。

#### 5.2.6 イベント登録（未来の資産）

ライブ・旅行・飲み会・デートなどの予定を登録すると「未来の心の資産」として表示される。

#### 5.2.7 理想バランス提案

AIが現在の年齢・ライフスタイルを考慮した理想的なポートフォリオ配分例を提示する。

```
理想バランス例
趣味  20%
友達  15%
家族  15%
恋人  20%
運動  10%
仕事  20%
```

## 6. 非機能要件

### 6.1 パフォーマンス

- ホーム画面の初期表示：2秒以内
- グラフ描画：1秒以内

### 6.2 セキュリティ

- 通信はHTTPS（TLS 1.2以上）
- パスワードはbcryptなどの強ハッシュでDB保存
- メンタルヘルス関連データは機密情報として扱い、第三者への提供は行わない
- データエクスポート・アカウント削除機能を提供（GDPR・個人情報保護法対応）

### 6.3 可用性

- 稼働率目標：99.5%以上

### 6.4 スケーラビリティ

- 初期リリース：〜1,000ユーザーを想定
- 将来的に10万ユーザーへのスケールアップを考慮した設計

### 6.5 アクセシビリティ

- フォントサイズ変更対応
- カラーバリアフリー（色覚特性に配慮したグラフカラー）

## 9. データモデル（概要）

```
User
├─ id
├─ email
├─ nickname
└─ created_at

Category
├─ id
├─ user_id
├─ name（恋人、筋トレ、など）
├─ parent_category（人、健康、など）
└─ is_custom（カスタム追加フラグ）

DailyRecord
├─ id
├─ user_id
├─ recorded_at（日付）
├─ category_id
├─ score（0〜100、マイナス可）
└─ note（任意テキスト）

WeeklyReport
├─ id
├─ user_id
├─ week_start_date
├─ summary_json（カテゴリ別集計）
└─ ai_comment
```
